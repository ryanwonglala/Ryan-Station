1. 项目概述 (Project Overview)

应用名称: 个人网站的动态明暗主题切换器。

核心目标: 在一个现有的index.html网站的侧边栏中，集成一个视觉效果独特的明暗模式切换开关。此功能需要能够根据用户的选择，流畅地改变整个网站的配色方案，并能记住用户的偏好设置。

目标用户: 网站的访问者。

关键特性列表:

在固定的侧边栏导航 <ul> 列表末尾，添加一个新的 UI 元素作为主题切换开关。

为该开关设计一个“太阳落下，月亮升起”的切换动画。

定义两种截然不同的配色主题：

明亮模式: 以米白色 (#F5F5DC) 为基调的柔和、清晰的风格。

黑暗模式: 以暗黑色 (#121212) 为基调的深邃、现代的风格。

确保主题切换能影响页面所有关键元素，包括背景渐变、侧边栏、文字颜色、图标颜色等。

使用浏览器的 localStorage 技术，实现主题偏好的持久化存储，用户下次访问时无需重新选择。

2. 模块逐步分解 (Step-by-step Module Breakdown)

模块一：HTML 结构修改

任务: 在 index.html 文件中，定位到 class 为 sidebar 的 <div> 内部的 <ul> 导航列表。

具体操作: 在最后一个包含 id="contact" 链接的 <li> 元素之后，但在 </ul> 闭合标签之前，插入以下 HTML 结构。这个结构将作为我们主题切换器的容器。

代码示例:

HTML
<li>
  <a href="#contact">
    <div class="icon"><i class="fas fa-envelope"></i></div>
    <div class="text">Contact</div>
  </a>
</li>

<li class="theme-switch-li">
  <div class="theme-switch-wrapper">
    <div class="theme-switch">
      <div class="sun"></div>
      <div class="moon"></div>
    </div>
  </div>
  <div class="text">Theme</div>
</li>
模块二：CSS 样式与主题化

任务: 使用 CSS 变量来定义和管理两种主题的配色方案，并为新的切换开关添加样式。

具体操作:

定义 CSS 变量: 在 <style> 标签的顶部，创建 :root 选择器来定义默认（黑暗模式）的颜色变量。然后，创建 body.light-mode 选择器来定义明亮模式下的颜色变量。

代码示例 (黑暗模式):

CSS
:root {
  --bg-gradient-start: #1c1c1c;
  --bg-gradient-mid: #121212;
  --bg-gradient-end: #000000;
  --sidebar-bg: rgba(18, 18, 18, 0.8);
  --sidebar-border: rgba(255, 255, 255, 0.1);
  --text-color: #e0e0e0;
  --icon-color: #e0e0e0;
  --hover-color: #ffa117;
  --section-text-color: rgba(255, 255, 255, 0.08);
  --active-shadow-color: rgba(18, 18, 18, 0.8);
  --transition-speed: 0.5s;
}
代码示例 (明亮模式):

CSS
body.light-mode {
  --bg-gradient-start: #F5F5DC; /* 米白色 */
  --bg-gradient-mid: #FAFAD2; /* 亮金黄色 */
  --bg-gradient-end: #FFF8DC; /* 玉米丝色 */
  --sidebar-bg: rgba(245, 245, 220, 0.7); /* 半透明米白色 */
  --sidebar-border: rgba(0, 0, 0, 0.1);
  --text-color: #3b3b3b;
  --icon-color: #2c2c2c;
  --hover-color: #f59e0b;
  --section-text-color: rgba(0, 0, 0, 0.08);
  --active-shadow-color: rgba(245, 245, 220, 0.7);
}
应用 CSS 变量: 审查整个 CSS 文件，将所有硬编码的颜色值（如 background, color, box-shadow 等）替换为上面定义的相应变量。例如，将 body { background: linear-gradient(...); } 修改为 body { background: linear-gradient(135deg, var(--bg-gradient-start), var(--bg-gradient-mid), var(--bg-gradient-end)); }。确保所有相关元素的颜色都由变量控制。

开关样式: 为 .theme-switch-li, .theme-switch-wrapper, .theme-switch, .sun, .moon 添加样式。

.theme-switch 应为一个圆角矩形容器。

.sun 是一个黄色的圆，并带有发光效果。

.moon 是一个白色的圆，并使用 inset box-shadow 技巧来创造月牙形状。

模块三：CSS 动画实现

任务: 创建太阳和月亮升起与落下的动画。

具体操作:

使用 transform: translateY() 和 transition 属性来控制太阳和月亮的位置和动画。

默认 (黑暗模式): 月亮在可见位置 (transform: translateY(0)), 太阳在开关下方隐藏 (transform: translateY(30px))。

明亮模式 (body.light-mode): 太阳移动到可见位置 (transform: translateY(0)), 月亮移动到开关上方隐藏 (transform: translateY(-30px))。

确保为 .sun 和 .moon 元素添加 transition: all var(--transition-speed) ease-in-out; 以实现平滑的动画效果。

模块四：JavaScript 交互逻辑

任务: 编写 JavaScript 代码来响应用户点击，切换主题，并保存用户选择。

具体操作:

获取元素: 使用 document.querySelector 获取 .theme-switch 开关和 document.body。

加载时检查: 页面首次加载时，编写一个函数来检查 localStorage.getItem('theme')。如果值为 'light-mode'，则立即给 body 添加 light-mode 类。

点击事件: 给 .theme-switch 添加一个 click 事件监听器。

切换函数: 在事件回调函数中：

使用 body.classList.toggle('light-mode') 来添加或移除 light-mode 类。

使用 if/else 语句检查 body 当前是否包含 light-mode 类。

如果包含，则 localStorage.setItem('theme', 'light-mode')。

如果不包含，则 localStorage.setItem('theme', 'dark-mode') 或 localStorage.removeItem('theme')。

3. 优先顺序 (Priority Order)

请按照以下顺序逐步实现功能，以确保逻辑清晰：

HTML 布局: 首先将开关的 HTML 结构添加到页面中。

CSS 变量化: 定义并替换所有颜色为 CSS 变量，并完成开关的静态样式。

JS 核心切换: 实现点击开关切换 body 上的 light-mode 类，并观察颜色变化是否生效。

CSS 动画: 在确认主题切换正常后，添加 transform 和 transition 来实现动画效果。

JS 持久化: 最后，添加 localStorage 逻辑，使主题选择能够被浏览器记住。

4. UI 设计 (UI Design)

组件位置: 固定在侧边栏导航列表的底部，与其他导航项对齐。

整体风格:

明亮模式: 整体感觉是温暖、柔和且易于阅读的。背景是多层次的米白色渐变，文字为深灰色，提供足够的对比度。

黑暗模式: 整体感觉是沉浸、专注且科技感十足的。背景使用几乎纯黑的深色渐变，避免使用纯黑以减少眼部疲劳，文字为浅灰色。

交互动画: 动画过程应平滑且直观，时长建议在 0.5 秒左右，使用 ease-in-out 缓动函数，给予用户自然、不突兀的视觉反馈。

5. 技术栈选择 (Technology Stack Selection)

前端: 原生 HTML, CSS, JavaScript (ES6)。

理由: 项目是基于一个独立的 HTML 文件，没有使用任何框架。因此，最直接、最高效的方案就是使用原生技术进行修改，无需引入任何外部库或依赖，保持项目的轻量和简洁。

后端/数据库: 不适用。状态持久化完全由客户端的 localStorage API 处理。
